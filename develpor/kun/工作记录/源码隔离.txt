# 可行方案
全源码工程，但排除资产目录。（起源方案）
大体思路如下：
	Repo划分
		1、Engine全工程库。
		2、Game工程库，排除Content目录。
		3、Content库。
		4、额外的runtime库
		5、拼一个 ProjectForDesigner 出来。
	采用cicd脚本将源码工程必要文件拷贝到内容工程。
	插件不做隔离，项目内代码使用UE的SudModule机制做区隔。

实践之后工程结构如下：
## 顶层项目工程
leeway(repo-root)
	project(repo-prg)
	doc(repo-doc)
	cicd(repo-cicd)

## 源码工程
project(repo-prg)
	engine(repo-engine)
	game(repo-game)
		Config
		Source
		Content(repo-Content)

## 内容工程
project_for_designer(repo-des)
	engine
		Engine
		Binaries
		Config
		Content
		Plugins
		Shaders
	game
		Config
		Source
		Content(repo-Content)
